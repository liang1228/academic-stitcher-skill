# Core Workflow

Run this workflow after loading the selected manifest fragments.

## 1. Intake

Capture target, deadline, venue level, advisor/graduation constraint, artifacts, and missing inputs. State the detected route and axes.

## 2. Paper Matrix

Before proposing modules, build or request a matrix of directly related papers: task, baseline, delta, bottleneck, mechanism, evidence, reusable piece, and risk.

## 3. Story Spine

Write the story sentence:

`This work inherits [baseline/task] from [paper family], but [failure mode] remains under [scenario], so it introduces [delta] to address [mechanism], supported by [evidence], within [boundary].`

## 4. Mechanism Test

Keep a proposed delta only if it has:

- a real failure mode;
- compatibility with the inherited baseline;
- a mechanism that predicts a measurable change;
- evidence that can be produced fairly.

## 5. Section Architecture

Map claims backward into title, abstract, introduction, related work, method, experiments, discussion, conclusion, and limitations. One paragraph has one job.

## 6. Draft Or Polish

Draft from evidence outward. For polishing, fix structure before sentences:

`paper type -> section job -> paragraph logic -> claim/evidence/boundary -> sentence polish`

## 7. Review

Run methodology, domain, skeptical-reviewer, and integrity checks. Preserve dissenting concerns until resolved by evidence or narrowed claims.

## 8. Checkpoint

Stop at the requested boundary: intake, outline, draft, audit, revision roadmap, or final check. Do not silently continue past the user's checkpoint.
