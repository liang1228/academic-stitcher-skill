# Section: Conclusion

## Job

Close the paper by stating what was shown, why it matters, and what remains limited.

## Structure

1. Restate the problem and contribution.
2. Summarize evidence, not every result.
3. State implication.
4. Name boundary or next step.

## Checks

- No new claims.
- No unsupported "first" or universal language.
- Match the target level: thesis, SCI, conference, or top venue.

## Abstract vs Conclusion: The Core Distinction (Video-Derived)

| Aspect | Abstract | Conclusion |
|--------|----------|------------|
| Scope | Introduction + Method (why + what) | Method + Experiments (what + how + results) |
| Opening | Field context and gap | Direct statement of what was done |
| Method detail | Name + high-level modules | What each module achieved |
| Evidence | Strongest single result | Summary of key findings |
| Future work | Omit or one sentence | 2-3 specific directions |

**Critical rule:** A+B module descriptions in the conclusion MUST be rephrased from the abstract. Identical sentences in both is a reviewer red flag.

### Conclusion Template

"In this paper, we proposed [Method Name] for [Task], integrating [Module B] and [Module C] to address [problems]. For [Module B], we [what it does and result achieved]. For [Module C], we [what it does and result achieved]. Experiments on [datasets] demonstrated [key results], validating each component. Future work will explore [direction 1], [direction 2], and [direction 3]."

### How to State Contributions Concisely

Restate contributions from the introduction, but with evidence rather than promise:

| Introduction (promise) | Conclusion (evidence) |
|------------------------|----------------------|
| "We propose Module B to address problem A" | "Module B achieves X% improvement on [problem A]" |
| "We design Module C for task Y" | "Module C outperforms baseline by X points on task Y" |
| "We integrate components for improvement" | "The framework achieves SOTA on [N] benchmarks" |

### Different Evidence State Strategies

| State | Strategy |
|-------|----------|
| Strong results | Lead with headline number; state contributions as validated; ambitious future work |
| Mixed results | Emphasize best-performing contribution; frame weaker as starting point |
| Weak results | Focus on methodological contribution; frame as preliminary; concrete future work |

### Common Mistakes

1. **Repeating abstract verbatim**: Must be written independently
2. **New results or claims**: No new numbers or experiments
3. **Too brief**: 3-5 sentences is too short; 1+ paragraph for conferences, 2-3 for journals
4. **Generic future work**: "We will continue to improve" -- specify WHAT and WHY
5. **Forgetting future work**: Most reviewers expect 2-3 concrete directions

### Chinese Academic Context

- Chapter 5 is "总结与展望" (Conclusion and Outlook): 5.1 summary + 5.2 outlook
- Chapter 5 summary MUST NOT repeat Chapter 1's research content using same sentences -- reviewers check
- Outlook: 3-4 specific directions, each tied to an experimental limitation
- Chinese thesis conclusions often include a "创新点总结" (innovation summary) as 3 bullet points
- Conclusion chapter in Chinese master's thesis: 2-5 pages
