# Section: Introduction

## Job
Move the reader from field pressure to the specific unresolved gap and present the study as the necessary next move.

## Default Funnel
Field pressure -> task/setting -> prior attempts -> unresolved technical challenge -> proposed move -> mechanism -> contribution preview.

## Variants
- Task then application. / Application first. / General task to specific setting.
- Open with technical challenge. / New task with decomposed challenges. / Pipeline or module contribution.

Tell the user which variant you selected when drafting.

## Funnel Structure with Sentence Starters (Video-Derived)

### Part 1: Field Description (1-2 paragraphs, no citations)
| Sentence Starter | Purpose |
|-----------------|---------|
| "In recent years, [task] has attracted increasing attention due to [application]." | Broad relevance |
| "[Domain] plays a critical role in [real-world scenario], enabling [benefit]." | Practical value |
> Source: BV1fW4y1W7dS, BV1dzNWewExA

### Part 2: Problem Introduction (with citations)
| Sentence Starter | Purpose |
|-----------------|---------|
| "However, existing methods typically suffer from [problem A], [problem B], and [problem C]." | Name the gaps |
| "As illustrated in Fig. X, current approaches fail to [specific failure]." | Visual evidence |

### Part 3: Proposed Solution
| Sentence Starter | Purpose |
|-----------------|---------|
| "To address these issues, we propose [Method Name], consisting of three components:" | State method |
| "1. We design [Module B] to handle [problem A] via [mechanism]." | Module link |
| "2. We introduce [Module C] to address [problem B] via [mechanism]." | Module link |
| "3. We integrate [Module D] to resolve [problem C] via [mechanism]." | Module link |

### Part 4: Result Preview
"Extensive experiments on [N] datasets demonstrate [X]% improvement in [metric]."
> Source: BV1qv4y1n7dx, BV1dzNWewExA, BV1ut34zuEfS (013)

### Building the "Field Pressure -> Gap -> Contribution" Chain
1. **Field pressure**: The field is growing, applications matter, problems exist
2. **Gap**: Methods fail at X, Y, Z ("find problems")
3. **Contribution**: Method addresses X, Y, Z through modules B, C, D
Problems MUST correspond to modules you use. B and C need exactly problems they solve.
> Source: BV1qv4y1n7dx, BV1npKfzPEuC (012)

### The "发现问题 -> 解决问题" Framing
- Frame as "I discovered these problems" NOT "I found modules to combine"
- Each problem needs a "therefore, we designed [module]" follow-up

### Different Evidence State Strategies
| State | Strategy |
|-------|----------|
| Strong contributions | Lead with strongest; include specific numbers |
| Incremental contributions | Frame as "extending"; emphasize practical value |
| Heavily borrowed method | Borrowed module first, novel module second |

### Common Mistakes
1. **No visual examples**: Show concrete problem examples with figures
2. **Problems don't match modules**: Claiming problem A but module solves B
3. **Too many citations in Part 1**: Field description should be citation-free
4. **Contribution list too long**: 3 ideal; 4 acceptable; 5+ is too many

## One-Sentence Argument

Every introduction must be anchored by one sentence:

> In [system/problem/domain], we show [advance] using [approach], supported by [evidence], with [boundary].

Every paragraph in the introduction must serve this sentence. If a paragraph does not advance toward this sentence, remove or relocate it.

### Chinese Academic Context
- Thesis introductions (Ch. 1) are 8-15 pages with "research significance"
- "国内外研究现状" mirrors introduction + related work combined
- Chinese journals often end with numbered "本文的主要贡献" list
